üìã Quick Start Guide

Follow these simple steps to get your app running:

1
Get the App Files

You should have received the app files after purchase. If not, please contact support.

You'll find these main folders:

  • meetlove_app - The main app for users
  • admin_panel - Web panel to manage your app
  • backend - The server code (already set up for you)

2
Set Up Your Server

We'll use Railway.app (free tier available) to host your app's server:

  1. Go to Railway.app and sign up
  2. Click "New Project" and select "Deploy from GitHub"
  3. Choose the backend folder from your files
  4. Railway will automatically set up your server
Save the website URL that Railway gives you - you'll need it later!
3
Set Up Firebase (For User Accounts)
  1. Go to Firebase Console
  2. Click "Add project" and follow the steps
  3. Click "Add app" and select Android
  4. Use your app's package name (example: com.yourcompany.meetlove)
  5. Download the google-services.json file
  6. Place this file in: meetlove_app/android/app/
Need help? Watch our video guide on setting up Firebase.

üéØ TourPlan Feature

Let users create and join local events and tours. No setup required - it works automatically!

What Users Can Do:

  • Create public or private tour plans
  • Set date, time, and location
  • Add photos and descriptions
  • Limit number of participants
  • Approve/decline join requests
  • Chat with tour participants
Automatic Setup: The TourPlan feature is already configured and ready to use. No additional setup needed!

Implementation Details

Database Schema
// TourPlan Model
{
  _id: ObjectId,
  title: String,
  description: String,
  creator: { type: Schema.Types.ObjectId, ref: 'User' },
  location: {
    type: { type: String, default: 'Point' },
    coordinates: [Number]
  },
  startDate: Date,
  endDate: Date,
  maxParticipants: Number,
  participants: [{
    user: { type: Schema.Types.ObjectId, ref: 'User' },
    status: { type: String, enum: ['pending', 'accepted', 'rejected'], default: 'pending' },
    joinedAt: { type: Date, default: Date.now }
  }],
  status: { type: String, enum: ['upcoming', 'ongoing', 'completed', 'cancelled'], default: 'upcoming' },
  createdAt: { type: Date, default: Date.now },
  updatedAt: { type: Date, default: Date.now }
}
API Endpoints
  • POST /api/tours - Create a new tour
  • GET /api/tours - Get all tours (with filters)
  • GET /api/tours/:id - Get tour details
  • POST /api/tours/:id/join - Request to join a tour
  • PUT /api/tours/:tourId/participants/:userId - Update participant status
  • DELETE /api/tours/:id - Cancel a tour

ü§ë Make Money with Ads (AdMob)

Earn money by showing ads in your app. Follow these simple steps:

5
Set Up Google AdMob
  1. Go to AdMob and sign in with your Google account
  2. Click "Add app" and select your app from the store (or create a placeholder)
  3. After adding your app, create these ad units:
    • Banner Ad (for the bottom of screens)
    • Interstitial Ad (full-screen ads that show between screens)
    • Rewarded Ad (users watch ads for bonuses)
  4. For each ad unit, copy the Ad Unit ID (it looks like ca-app-pub-xxxxxxxxxxxxxxxx/yyyyyyyyyy)
Update App with Your Ad IDs

Open the file: meetlove_app/lib/config/ads_config.dart

Replace the placeholder text with your AdMob IDs:

class AdConfig {
  static const String appId = 'YOUR_ADMOB_APP_ID';
  static const String bannerAdId = 'YOUR_BANNER_AD_ID';
  static const String interstitialAdId = 'YOUR_INTERSTITIAL_AD_ID';
  static const String rewardedAdId = 'YOUR_REWARDED_AD_ID';
}
The app is already set up to show ads in the best places for user experience and maximum earnings.

Removed Features

The following features have been removed in version 1.1:

1. Booking System

The previous booking system has been completely removed and replaced with the new TourPlan feature.

2. Stripe Payment Integration

Stripe payment processing has been removed. The app now uses in-app purchases for premium subscriptions.

3. Deprecated APIs

The following API endpoints have been removed:

  • POST /api/bookings
  • GET /api/bookings
  • POST /api/payments/process
  • GET /api/payments/history

Enhanced User Discovery

The user discovery system has been completely revamped with new filtering options and improved matching algorithms.

New Discovery Features
