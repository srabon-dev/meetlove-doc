Introduction

Thank you for your interest in MeetLove!

MeetLove is a complete Modern Dating Platform built with the Flutter Framework, designed for Android and iOS devices, along with a powerful Web Admin Dashboard and a secure Node.js backend.

Using Google's open-source Flutter technology, MeetLove offers a high-performance, cross-platform experience with real-time chat, TikTok-style video feed, random live streaming, partner booking system, secure Stripe payments, and more ‚Äî all crafted to deliver the ultimate modern dating experience.

2. Stripe Account Setup

To enable payments on MeetLove, you need to set up a Stripe account for payment processing. Follow these steps to create a Stripe account and get your live API keys:

  • Step 1: Create a Stripe Account
    • Go to Stripe Registration to sign up for a Stripe account.
    • Fill in the required details such as email, business information, and create a password.
    • Complete the verification process (this may include verifying your email address and business details).
    • Once your account is created, you‚Äôll be redirected to the Stripe Dashboard.
  • Step 2: Get Live API Keys
    • In the Stripe Dashboard, go to the Developers section in the left-hand menu.
    • Click on API keys to access your live and test API keys.
    • You will see two types of keys:
      • Publishable Key: This key is used on the client side (Flutter app) to initiate payments.
      • Secret Key: This key is used on the server side (backend) to handle the payment processing securely.
    • Make sure you're in the "Live" mode to get live keys. The keys will look like this: 
      sk_live_xxxxxxx
      (for the secret key) 
      pk_live_xxxxxxx
      (for the publishable key)
    • Copy both the secret and publishable keys for use in your backend and Flutter app.
  • Step 3: Integrating the Stripe Keys in Your Backend
    • In your backend (Node.js), you need to add the Stripe API keys to your environment variables for security. Open your `.env` file (make sure it's in your project‚Äôs root directory).
    • Add the following lines to your `.env` file: 
      stripeSecretKey=sk_live_xxxxxxx
    • Replace sk_live_xxxxxxx and pk_live_xxxxxxx with the actual keys you obtained from the Stripe Dashboard.
    • This ensures that your Stripe keys are securely stored and not exposed in your source code.

6. MongoDB Atlas Setup (For Database)

MongoDB Atlas is a fully-managed cloud database service. Follow these steps to set up MongoDB Atlas and connect it to your Node.js backend:

  • Go to MongoDB Atlas and sign up for an account.
  • Create a new cluster for your project:
    • Click on the Build a Cluster button on the MongoDB Atlas dashboard.
    • Choose your cloud provider and region (e.g., AWS, GCP, or Azure).
    • Select the free tier (M0) or any other tier based on your requirements.
    • Click on Create Cluster and wait for the cluster to be provisioned. This may take a few minutes.
  • After your cluster is ready, you need to create a database:
    • In the Clusters section, click on the Collections button for your cluster.
    • Click on Create Database to set up a new database.
    • Give your database a name (e.g., "meetlove") and add a collection (e.g., "users").
  • Now, configure the database security settings:
    • Click on the Database Access tab in the left sidebar under the "Security" section.
    • Click on + ADD NEW DATABASE USER to create a new user for your application.
    • Enter a username and password for the new user. Ensure you choose the correct database and give appropriate permissions, such as "Read and Write to any database" if required.
    • Click Add User to create the user.
  • Set up IP Whitelisting to allow your application to connect:
    • Go to the Network Access tab under the "Security" section.
    • Click on + ADD IP ADDRESS to allow specific IP addresses or click on Allow Access from Anywhere to permit connections from any IP (not recommended for production).
  • Get the connection string (MongoDB URL):
    • Go to the Clusters tab.
    • Click on Connect for your cluster.
    • Choose Connect Your Application and select your connection method (e.g., Node.js).
    • Copy the provided connection string. It will look like this:
      • mongodb+srv://:@cluster0.mongodb.net/?retryWrites=true&w=majority
    • Replace and with the credentials you just created, and replace with the database name (e.g., "meetlove").
  • Integrate the MongoDB connection string into your Node.js backend:
    • In your backend project, open the .env file (or create it if it doesn't exist).
    • Add the MongoDB connection string as a variable in the .env file:
      •       MONGO_URL="mongodb+srv://:@cluster0.mongodb.net/meetlove?retryWrites=true&w=majority"

  • Set up environment variables for your project:
    • After deployment, you can add environment variables by going to the Environment Variables section in Railway.
    • Add any necessary variables like MONGO_URL for MongoDB, STRIPE_SECRET_KEY for Stripe, or any other keys you need to keep secure.
  • Ensure that your server is running properly by checking the logs in Railway:
    • Click on the Logs tab to monitor your server‚Äôs status and view any error messages.
  • Important: Once your project is complete and you have uploaded it to GitHub, make sure that your server is deployed and functioning properly on Railway before you proceed with the Flutter web and Node.js backend deployment.
  • Notice for Flutter Web and Node.js Backend:
    • Once your Node.js backend and Web Admin Dashboard are successfully deployed to Railway, the backend should be accessible via a URL provided by Railway.
    • In your Flutter web app, ensure that your API calls are directed to the Railway-hosted backend instead of local endpoints (e.g., replace `http://localhost:5000` with the Railway URL).
    • Test the integration thoroughly to ensure that the Flutter web app communicates correctly with the deployed backend server.

Common Issues in Flutter App & Node.js Backend

1. Flutter App: Dependency Issues ▶

Issue: "flutter pub get" fails due to conflicting versions or missing dependencies.

Solution: Run the following commands to resolve dependencies:

  • flutter clean - Clears build cache.
  • flutter pub upgrade - Updates all dependencies to the latest versions.
  • If issues persist, manually resolve version conflicts in pubspec.yaml.

2. Node.js Backend: MongoDB Connection Error ▶

Issue: "MongooseError: connect ECONNREFUSED" or other MongoDB connection errors.

Solution: Check the following:

  • Ensure the MongoDB Atlas URL in your .env file is correct.
  • Verify the database is running and accessible.
  • Check your MongoDB Atlas IP whitelist to allow connections from your server's IP.

3. Flutter App: App Crashes on Launch ▶

Issue: Flutter app crashes on launch with no clear error.

Solution: Check the following:

  • Run the app with flutter run --verbose to get more detailed logs.
  • Ensure that your flutter doctor shows no errors.
  • If the issue is with iOS, ensure that the Info.plist file is correctly configured.

4. Node.js Backend: CORS Issues ▶

Issue: "CORS policy: No 'Access-Control-Allow-Origin' header is present on the requested resource."

Solution: Make sure that you have properly set up CORS in your Node.js backend:

  • Install the cors package: npm install cors
  • In your app.js or main backend file, add the following:
    •               const cors = require('cors');
              app.use(cors({
                origin: ['http://localhost:3000', 'https://your-app-url.com'],
                credentials: true
              }));

5. Node.js Backend: Express App Not Responding ▶

Issue: Express app is not responding or taking too long to process requests.

Solution: Check the following:
